## Overview

**EvalEmulateSurv** is an R package for evaluating how well emulated
individual patient data (IPD) reproduces the original IPD with respect to
survival endpoints.

Given a pair of datasets -- an original IPD and an emulated IPD generated by
any method -- the package computes three complementary metrics between their
Kaplan-Meier survival curves:

| Metric | Measures |
|--------|----------|
| **NAUC** | Average absolute deviation over a truncation window |
| **KS distance** | Maximum (worst-case) absolute deviation |
| **RMSE** | Outlier-sensitive deviation over the full observation range |

All metrics can be computed for the overall population and for subgroups
defined by single variables or N-way interaction terms (e.g., `"SEX:REGION"`).
Descriptive survival statistics (sample size, event count, median survival
time with confidence interval) are also provided alongside the metrics.

## Installation

```r
# Install from GitHub (requires the remotes package)
remotes::install_github("gohomma/EvalEmulateSurv")
```

## Main functions

| Function | Description |
|----------|-------------|
| `load_ipd()` | Load IPD from a `data.frame`, tibble, `.csv`, or `.RData` file |
| `km_est()` | Kaplan-Meier estimation with Greenwood variance, pointwise CI, and descriptive statistics |
| `nauc()` | Normalised Area Under the Curve |
| `ksdist()` | Kolmogorov-Smirnov distance |
| `rmse()` | Root Mean Square Error |
| `kmplot()` | KM plots with metric annotations |
| `summarytable()` | Summary table of descriptive statistics and all three metrics by subgroup |
| `gen_dummy_data()` | Generate dummy IPD for examples and testing |

## Quick start

```r
library(EvalEmulateSurv)

# Generate dummy data
ipd <- gen_dummy_data(seed = 42)

# Split by endpoint and group
ipd_os  <- ipd[ipd$TYPE == "OS", ]
grp1_os <- ipd_os[ipd_os$GROUP == "Drug",    ]
grp2_os <- ipd_os[ipd_os$GROUP == "Control", ]

# Summary table: overall + subgroups
res <- summarytable(
  data_group1   = grp1_os,
  data_group2   = grp2_os,
  time_var      = "TIME",
  event_var     = "EVENT",
  tau           = 36,
  subgroup_vars = c("SEX", "REGION", "SEX:REGION"),
  label_group1  = "Original",
  label_group2  = "Emulated"
)
print(res)

# KM plot with subgroup panels
plots <- kmplot(
  data_group1  = grp1_os,
  data_group2  = grp2_os,
  time_var     = "TIME",
  event_var    = "EVENT",
  tau          = 36,
  subgroup_var = "REGION",
  ncol         = 3
)
plots[["REGION"]]
```

## Input data format

Each function accepts `data_group1` and `data_group2` as:

- A `data.frame` or tibble already in memory
- A path to a `.csv` file
- A path to a single-object `.RData` / `.rda` file

For multi-endpoint datasets (e.g., OS and PFS in the same data frame), filter
to one endpoint before passing to any function:

```r
ipd_os <- ipd[ipd$TYPE == "OS", ]
```

## KM estimation details

`km_est()` is implemented from scratch without relying on the `survival`
package. It returns:

- KM survival estimates (`surv`) per group
- Greenwood variance (`var_surv`) and pointwise confidence bands
  (`surv_lo`, `surv_hi`) using the log transformation -- equivalent to
  `survival::survfit(conf.type = "log")`
- Descriptive statistics per group (`desc1`, `desc2`): sample size, event
  count, event rate, median survival time and its confidence interval
